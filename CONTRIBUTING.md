# Beitragen zu Tuizz 💃🕺

Vielen Dank, dass Sie sich für Tuizz interessieren! Wir freuen uns über jede Art von Beitrag – von Fehlerberichten über Funktionsvorschläge bis hin zu Code-Verbesserungen.

## Wie kann ich beitragen?

### 🐛 Fehler melden
Wenn Sie einen Fehler finden, erstellen Sie bitte ein Issue auf GitHub. Beschreiben Sie das Problem so detailliert wie möglich, inklusive Schritten zur Reproduktion.

### ✨ Funktionsvorschläge
Haben Sie eine Idee für ein neues Feature? Wir hören sie gerne! Erstellen Sie ein Issue und markieren Sie es als "Feature Request".

### 💻 Code-Beiträge
1. Forken Sie das Repository.
2. Erstellen Sie einen neuen Branch für Ihre Änderungen (`git checkout -b feature/mein-neues-feature`).
3. Nehmen Sie Ihre Änderungen vor.
4. Committen Sie Ihre Änderungen (`git commit -m 'Füge neues Feature hinzu'`).
5. Pushen Sie den Branch (`git push origin feature/mein-neues-feature`).
6. Erstellen Sie einen Pull Request.

## Code-Stil
- Wir verwenden **Tailwind CSS** für das Styling.
- JavaScript sollte klar und gut kommentiert sein.
- Halten Sie die Anwendung so leichtgewichtig wie möglich (keine unnötigen Abhängigkeiten).

## Verhaltenskodex
Bitte beachten Sie unseren [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md). Wir legen Wert auf eine freundliche und inklusive Community.
